KogniTerm Architecture

AI-powered terminal with multi-provider LLM integration, skill system, and TUI

Terminal Layer

  • • cli.py: CLI entry point (config, keys, index, models commands)
  • • terminal.py: Async orchestrator, dotenv config injection
  • • KogniTermTUI: Textual-based TUI with components
  • • TerminalUI: Rich rendering and streaming output
  • • CommandApprovalHandler: Interactive command approval
  • • MetaCommandProcessor: @ slash command processing
  • • AgentInteractionManager: Multi-agent coordination

Core Layer

  • • LLMService: Central hub - LiteLLM, streaming, tool parsing
  • • MultiProviderManager: OpenRouter, Gemini, Ollama, Anthropic
  • • SessionManager: JSON session save/load/list
  • • HistoryManager: Token tracking, message pruning
  • • CommandExecutor: Shell command execution
  • • AgentState: Loop detection, message accumulation
  • • Specialized agents: CodeAgent, DeepCoder, DeepResearcher, BashAgent

Skills System

  • • SkillManager: Discovery, JIT loading, tool registry
  • • bundled/: call_agent, execute_command, file_operations, memory_*, web_tools, code_tools
  • • workspace/: photo_organizer, project_analyzer, docker_error_logs
  • • managed/: User skills at ~/.kogniterm/skills/managed/
  • • Each skill: SKILL.md (YAML frontmatter) + scripts/*.py
  • • SkillValidator: Schema and security validation
  • • SkillLoader: Dynamic module import with metadata injection

Data Layer

  • • VectorDBManager: ChromaDB (SQLite) - codebase_chunks collection
  • • CodebaseIndexer: AST parsing, code chunking for embeddings
  • • ProjectMemoryBuilder: Workspace context aggregation
  • • EmbeddingsService: Cohere embeddings for semantic search
  • • Storage: .kogniterm/vector_db/, .kogniterm/sessions/

KogniTerm • Python + Textual TUI + LiteLLM + LangChain + ChromaDB
